## Fixed an issue where the device's lock status could become stuck after a failed lock or unlock operation

In rare cases, when a remote lock or unlock operation failed and the user chose not to retry via Bluetooth, the lock status in the app could become stuck in a "moving" state. This prevented the user from performing any further lock operations until they closed and reopened the application.

With this fix, when a remote operation fails and the user dismisses the retry option, the app now correctly updates the lock status from the server. This ensures that you can continue operating your lock without needing to restart the application.
