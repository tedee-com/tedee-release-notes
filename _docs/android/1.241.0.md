## Widgets now work without live lock state
Android widgets are now fully functional even when the lock state is not actively being refreshed. Widget interactions no longer require a live state refresh to operate, providing a more seamless experience.

## Open lock details from widget
You can now tap a lock name directly in the devices home screen widget to navigate immediately to that lock's details page, eliminating the need for additional navigation steps.

## Open WebView pages in browser
Pages rendered in the in-app WebView now include a dedicated button that allows you to open them in your external browser for a full browsing experience.

## Fixes
- Fixed a fatal crash in Auto Unlock that occurred during foreground notification setup.
- imroved auto-unlock — the feature now checks RSSI level and will not connect to the lock if the signal strength is poor.
- Fixed an issue with lock unlocked notifications — the app now correctly respects location accuracy in distance calculations.
